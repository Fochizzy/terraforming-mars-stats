# Deployment Notes

## Runtime env in Cloudflare

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `SUPABASE_STORAGE_BUCKET_IMPORT_EVIDENCE`
- `SUPABASE_SERVICE_ROLE_KEY` for exact web import participant-set matching

The authenticated web import flow now requires `SUPABASE_SERVICE_ROLE_KEY` in the deployed Worker so the server can inspect existing cross-group rosters while resolving or creating the target group.

## Production domain

- Primary URL: `https://tm-stats.com`
- Worker custom domains: `tm-stats.com` and `www.tm-stats.com`
- Cloudflare DNS zone nameservers: `ashley.ns.cloudflare.com` and `kip.ns.cloudflare.com`

The domain must be active in Cloudflare before the Worker custom domains can be attached. The `wrangler.jsonc` routes keep the custom-domain mapping declared for future deploys.

## Supabase Auth and Resend email

- Supabase Auth Site URL: `https://tm-stats.com`
- Supabase Auth Redirect URLs:
  - `https://tm-stats.com/auth/callback`
  - `https://www.tm-stats.com/auth/callback`
  - `https://terraforming-mars-stats.izzy-hodnett.workers.dev/auth/callback` during the transition
- Resend sending domain: `mail.tm-stats.com`
- Supabase SMTP settings:
  - Host: `smtp.resend.com`
  - Port: `465`
  - Username: `resend`
  - Password: the Resend API key
  - Sender email: `noreply@mail.tm-stats.com`
  - Sender name: `Terraforming Mars Stats`

Add the DNS records generated by Resend for `mail.tm-stats.com` in Cloudflare DNS, then verify the domain in Resend before enabling Supabase SMTP in production.

## Daily catalog refresh

`.github/workflows/tfm-card-source-check.yml` scans the Terraforming Mars card
browser at `https://terraforming-mars.herokuapp.com/cards#~trbgpcseCmalt` every
day and can also be run manually from GitHub Actions. The workflow fetches the
live card source, publishes cards/corporations/preludes to Supabase, force
renders card images and thumbnails into Supabase Storage, and finishes with a
dry-run coverage check.

Required GitHub Actions secret:

- `SUPABASE_SERVICE_ROLE_KEY`

The public Supabase project URL is set directly in the workflow as
`https://qjtwgrjjwnqafbvkkfex.supabase.co`.

## Launch order

1. Create the new Supabase project.
2. Apply the repo migrations with `npx.cmd supabase db push`.
3. The seed migration now loads the stable expansion, map, and style dimensions automatically when you push schema changes.
4. Run `npm.cmd run catalog:publish` to create a catalog snapshot row and upsert the Terraforming Mars card-browser cards, corporations, preludes, tags, and promo-set links.
5. Add the remaining corporation, prelude, promo-set, milestone, award, and card imports before inviting the first group.
6. Set the Cloudflare runtime env vars and deploy the Worker.
7. Activate `tm-stats.com` in Cloudflare, then attach `tm-stats.com` and `www.tm-stats.com` to the Worker.
8. Add and verify the Resend `mail.tm-stats.com` DNS records in Cloudflare.
9. Set Supabase Auth Site URL to `https://tm-stats.com`, add the `/auth/callback` redirects, and enable Resend SMTP.
10. Run `npm.cmd run test`, `npm.cmd run build`, and `npx.cmd opennextjs-cloudflare build` before calling the deployment path ready.
