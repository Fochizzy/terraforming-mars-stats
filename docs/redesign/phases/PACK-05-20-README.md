# TM Stats Expanded Phase Markdown Pack — Phases 5–20

## Authority and purpose

These files are expanded execution versions of Phases 5–20 from the integrated Word implementation guide. The Word guide remains the source of truth. The expansion does **not** reorder, replace, or add source implementation steps; it adds nested stages so each source step can be inspected, implemented, tested, committed, and handed off as a bounded task.

The structure follows the detailed Phase 4 pattern: status, source authority, prerequisites, protected boundaries, a phase working sequence, stage-by-stage execution, validation gates, acceptance criteria, and immutable handoff requirements.

## Use rule

Give an agent only one explicitly assigned source step or Phase 6 substep at a time. The agent must stop after validation, documentation, commit, and handoff. The presence of the remaining files in this pack is not authorization to begin them.

- Every phase title, recommended agent table, outcome, rationale, prerequisite, inspection list, prohibition, source step, copy-ready prompt, acceptance checklist, and required handoff from the Word guide is retained.
- Source step order is retained exactly.
- Phase 20 retains the Word guide’s unnumbered **Validate win point differential analytics** heading between source steps 7 and 9 rather than silently renumbering it.
